# Makefile Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Makefile with `download-signal-cli` and `run` targets, and gitignore the `bin/` directory where the binary lands.

**Architecture:** Two-file change — append `bin/` to `.gitignore`, then create `Makefile` at repo root. `download-signal-cli` fetches the tarball to `/tmp`, extracts into `bin/` with `--strip-components=1`, and cleans up. `run` prepends `./bin` to `PATH` before invoking the agent.

**Tech Stack:** GNU Make, wget, tar

### Task 2: Create `Makefile`

**Files:**
- Create: `Makefile`

- [ ] **Step 1: Create the Makefile**

```makefile
.PHONY: download-signal-cli run

SIGNAL_CLI_VERSION = 0.14.5
SIGNAL_CLI_URL     = https://github.com/AsamK/signal-cli/releases/download/v$(SIGNAL_CLI_VERSION)/signal-cli-$(SIGNAL_CLI_VERSION)-Linux-client.tar.gz

download-signal-cli:
	mkdir -p bin
	wget -O /tmp/signal-cli.tar.gz $(SIGNAL_CLI_URL)
	tar -xzf /tmp/signal-cli.tar.gz -C bin --strip-components=1
	rm /tmp/signal-cli.tar.gz

run:
	PATH=./bin:$$PATH python -m summarizer_agent.main
```

> **Note:** Lines under each target MUST be indented with a real tab character, not spaces. Most editors convert tabs to spaces — verify with `cat -A Makefile` and confirm `^I` appears before each recipe line.

- [ ] **Step 2: Verify tab indentation**

```bash
cat -A Makefile | grep -E "^\^I|^I"
```

Expected: lines starting with `^I` (tab) before `mkdir`, `wget`, `tar`, `rm`, and `PATH=`.

- [ ] **Step 3: Smoke-test `download-signal-cli`**
